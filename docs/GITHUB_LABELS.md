# GitHub Labels

Create these labels before enabling public issue intake. They match the issue
forms in `.github/ISSUE_TEMPLATE/`.

| Label | Used by | Meaning |
|---|---|---|
| `bug` | Bug report | Crash, install failure, wrong diagnosis, or other unexpected behavior. |
| `board-data` | Board report | Keyboard model metadata, board-type hints, or repair-ladder corrections. |
| `enhancement` | Feature or diagnostic request | Scoped diagnostic, workflow, packaging, app, docs, or repair-guide improvement. |
| `hardware-smoke` | Manual smoke report | Real Windows keyboard smoke evidence for release gating. |
| `good first issue` | Maintainer-seeded starter issues | Small, evidence-backed tasks with clear local proof and no unsupported claims. |
| `help wanted` | Maintainer-seeded public contribution lanes | Board data, docs, tests, or smoke evidence where outside input is useful. |

Do not add labels that imply unsupported scope such as cross-platform support,
remapping, macros, latency certification, or broad hardware proof.
